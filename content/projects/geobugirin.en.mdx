## Project Overview

Geobugirin is a "set-and-forget" desktop application for posture correction. It addresses the forward head posture problem faced by developers who sit at computers all day, while fully respecting user privacy.

## Key Features

<FeatureList icon="check">
  <FeatureItem>Complete privacy protection - all processing happens locally</FeatureItem>
  <FeatureItem icon="eye">Real-time posture estimation with MediaPipe Pose</FeatureItem>
  <FeatureItem icon="zap">Non-intrusive always-on-top widget</FeatureItem>
  <FeatureItem icon="bell">OS native notification system integration</FeatureItem>
  <FeatureItem icon="shield">Zero video data transmission to server</FeatureItem>
</FeatureList>

## Core Features

### 1. Privacy-First Design

Geobugirin ensures that no camera video or image data is ever transmitted to external servers. All posture detection and analysis happens entirely on the user's computer.

<TwoColumn>
  <div>
    **Design Principles**

    - No video/image data server transmission
    - All decision logic executed client-side
    - Optional summary statistics only
    - User never feels "monitored"
  </div>

  <div>
    **Technical Implementation**

    - MediaPipe Pose for local inference
    - Geometry-based posture determination
    - Server stores summary statistics only
    - Client-only processing without WebSocket
  </div>
</TwoColumn>

### 2. On-Device AI

Uses MediaPipe Pose for real-time pose estimation in browser and desktop environments.

<TwoColumn>
  <div>
    **MediaPipe Utilization**

    - 33 joint landmark tracking
    - Real-time coordinate returns
    - Lightweight computational cost
    - Stable human body recognition
  </div>

  <div>
    **Posture Determination**

    - Neck tilt calculation
    - Upper body angle analysis
    - Good/bad posture classification
    - Rule-based determination system
  </div>
</TwoColumn>

### 3. UX: Non-Intrusive Experience

<ProblemSolution>
  <Problem title="Browser Tab Limitations">
    Web environments have limited system-level control. Always-visible UI, non-interrupting notifications, and long-running stability are difficult to achieve with browser-only solutions.
  </Problem>

  <Solution title="Electron Desktop App">
    Chose Electron to maintain React development experience while providing native-like UX. Implemented always-on-top widget and OS native notifications for a service users can rely on without conscious thought.
  </Solution>
</ProblemSolution>

## Tech Stack

### Frontend
- **Electron**: Desktop app framework
- **React**: UI components
- **TypeScript**: Type safety

## Achievements

- 500+ downloads per month
- Average usage time: 4 hours/day
- Zero privacy-related complaints
- User satisfaction: 4.5/5

## Key Learnings

1. **On-Device AI tradeoffs**: Server cost savings vs accuracy limitations
2. **Privacy-centered design**: User trust > technical perfection
3. **Electron + React synergy**: Extending web to desktop
4. **Minimal UX philosophy**: Non-intrusive is the best feature
