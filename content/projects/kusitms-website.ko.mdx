## 프로젝트 개요

![웹사이트 현대화](https://images.unsplash.com/photo-1517292987719-0369a794ec0f?w=1200&h=600&fit=crop)

고려대학교 소프트웨어 동아리 KUSITMS의 공식 웹사이트를 Astro와 React를 사용하여 현대화하고 리브랜딩하는 프로젝트입니다. 3단계 스프린트로 진행되며, 코드 리팩토링, 디자인 시스템 구축, GA 연동 등을 포함합니다.

### 2. 성능 최적화

<ProblemSolution>
  <Problem title="이미지 최적화 문제">
    Vercel에서 제공하는 이미지 최적화 할당량을 초과하면 이미지가 화면에 전혀 노출되지 않는 문제가 발생했습니다. 31기 웹 TF 진행할 때도 동일한 문제가 있었습니다.
  </Problem>

  <Solution title="직접 이미지 최적화 구현">
    Next.js의 Image 태그를 사용하지 않고 직접 이미지 최적화 기능을 구현했습니다. 다양한 이미지 최적화 방법 중 하나를 선택하여 일관되게 적용하고, a 문제가 발생한 원인과 해결 방법을 포트폴리오에 작성했습니다.
  </Solution>
</ProblemSolution>

## 기술 스택

### Frontend
- **Astro**: 메인 프레임워크
- **React**: 인터랙티브 컴포넌트
- **TypeScript**: 타입 안전성
- **Vite**: 빌드 도구

### Styling
- **Tailwind CSS**: 유틸리티 CSS 프레임워크
- **Atomic Design**: 컴포넌트 설계 시스템

### Analytics
- **Google Analytics**: 방문자 행동 추적

### Deployment
- **Vercel**: 호스팅 플랫폼

## 기술적 도전 과제

<ProblemSolution>
  <Problem title="아일랜드 아키텍처 설계 복잡성">
    Astro의 아일랜드 아키텍처를 적용하면서 정적 콘텐츠와 클라이언트 인터랙티브 컴포넌트를 적절히 분리하는 것이 어려웠습니다.
  </Problem>

  <Solution title="명확한 분리 계층 구조">
    - 정적 페이지는 Astro에서 SSR 처리
    - 인터랙티브 컴포넌트는 React로 클라이언트 로드
    - 프로젝트 구조를 명확하게 분리하여 유지보수성 확보
  </Solution>
</ProblemSolution>
